资源

1.EG(regular_list) 存储着所有的只用在当前请求的{资源}

2.EG(persistent_list) 存储着所有的持久{资源}

3.对于持久资源，我们打开一个本地文件，就可以一直占用它的操作句柄，随时可以进行相应的只得。但如果是远程计算机机上的资源，如mysql链接，http链接，虽然我们仍然有与服务器的链接，但是服务器端有可能关闭了，那么这样的持久链接就没有价值了。如何检测持久资源的有效性呢？如果无效了，我们需要将其从EG(persistent_list) HashTable中删除

流、流的类型和流包装层

Streams were introduced with PHP 4.3.0 as a way of generalizing file, network, data compression, and other operations which share a common set of functions and uses. A wrapper is additional code which tells the stream how to handle specific protocols/encodings.

stream实质也是封装了的PHP资源类型，当创建流时自动的往内核中注册了相应的资源类型。

PHP中的流在php-4.3.0引入的，通常直接的文件描述符就可以完成PHP中流的功能，但是为了完成一个扩展实现特定的协议，我们会自己实现诸如ssl、http等。庆幸的是PHP引入了流及流的包装器可以方便扩展实现特定的协议，并在用户空间提供一套标准的函数。

通过流包装层, 扩展代码可以透明的使用各种内建的流包装, 比如HTTP, FTP, 以及它们对应的SSL版本, 另外还有gzip和bzip2压缩包装. 通过include特定的PEAR或PECL模块,还可以访问其他协议, 比如SSH2, WebDav, 甚至是Gopher!

PHP内核流与流包装器的关系图

PHP内核中关于php_stream,main/php_stream.h中有这样的定义：

php_stream重要结构体成员描述：

php_stream_ops *ops：流访问的函数指针集合，如write,read

void *abstract：便利指针，指向我们捆绑自定义实现的数据结构，通过这个指针可以获取到所有捆绑的信息

php_stream_filter_chain readfilters, writefilters： 流过滤器链表

php_stream_ops重要结构体成员描述：

size_t (*write)：写入数据流的函数指针

size_t (*read)：读取数据流的函数指针

int (*seek)：随机访问时的设置流的内部指针位置的函数指针

php_stream_wrapper_ops重要结构成员描述：

php_stream (stream_opener)：创建流函数指针

int (*stream_closer)：关闭流函数指针

各种流的实例，文件流或者网络流，它们不同在于流创建函数返回的php_stream结构中的ops成员。

当流访问函数比如php_stream_read()被调用时, 流包装层实际上解析调用了stream->ops中对应的函数, 这样实际调用的就是当前流类型特有的read实现

ops结构体指向的函数指针常常是对数据源真正的读取函数的一个瘦代理

关于stat()和lstat()的区别, 查看*nix手册
