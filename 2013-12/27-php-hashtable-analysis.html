PHP内核之HashTable实现分析与使用

发表于 2013-12-02   |   分类于 PHP   |  

HashTable

即哈希表，又名散列表，是根据关键字（key value)直接访问在内存存储位置的数据结构。是一种将随机访问的数据结构转换为有序储存的数据结构。它通过把键值通过一个函数的计算，映射到表中一个位置来访问，此函数称为散列函数。现在很多高级程序语言中都有实现HashTable，而C语言中最常用的是uthashtable。

PHP内核中HashTable（以php-5.4为例）

PHP作为C语言的开源应用，其HashTable的实现既经典又是创新的。PHP中的HashTable实现代码保存在Zend/zend_hash.h和Zend/zend_hash.c这两个文件中，首先来分析其结构体的定义：

在上面的结构定义中,Bucket与HashTable这两个结构体构成一个完整的HashTable

HashTable结构体成员说明

uint nTableSize:记录Bucket数组的大小，即成员Bucket **arBuckets的大小

uint nTableMask:掩码，值为nTableSize-1，经过hash函数计算的hash值对掩码作取模运算，取得储存数据的位置

uint nNumOfElements:已储存在HashTable中的元素个数

ulong nNextFreeElement:下一个可用的Bucket位置，一般在数字索引的HashTable中有用

Bucket *pInternalPointer:遍历HashTable内部实现中使用

Bucket *pListHead:指向HashTable实现中的双向链表的表头，采用双向链表的数据结构，主要为了PHP中有序遍历HashTable中的元素

Bucket *pListTail:指向HashTable实现中的双向链表的表尾

Bucket **arBuckets:Bucket桶数组，HashTable中所有的元素都储存在这个二级指针数组中，每个元素是一个指向Bucket结构体的指针，每个Bucket代表一个元素

dtor_func_t pDestructor:析构函数指针，当释放HashTable中储存的元素时，需要执行的析构函数

zend_bool persistent：永久性内存分配标识

unsigned char nApplyCount:暂时不知其应用场景

zend_bool bApplyProtection:暂时不知其应用场景

int inconsistent:调试相关的成员，暂时不知其应用场景

Bucket结构体成员说明

ulong h:保存经过hash函数处理之后的值

uint nKeyLength:字符串key的长度

void *pData:指向要保存的内存块地址

void *pDataPtr:用来保存指令数据，如果要保存的数据是指针时，直接保存到该指针，然后pData指向pDataPtr

struct bucket *pListNext:指向双向链表的下一个元素

struct bucket *pListLast:指向双向链表的上一个元素

struct bucket *pNext:指向具有同一个hash值的下一个元素

struct bucket *pLast:指向具有同一个hash值的 上一个元素

const char *arKey:字符串key，这里用到了C语言的灵活数组成员的特性（Flexible Array）

需要特殊说明的是，zend_hash_init的每个参数

第一个参数：HashTable *

第二个参数：uint,初始化HashTable的大小，但真实分配的大小不一定为此参数指定的值，PHP内核为计算出一个不小于它且为最小的2的次方值

第三个参数：hash_func_t,此类型为指向hash函数的函数指针，但现已废弃故这里为NULL

第四个参数：dtor_func_t，此类型为指向元素析构函数的函数指针这里为ZVAL_DESTRUCTOR，表示我们将在HashTable中储存zval *的数据地址

第五个参数：zend_bool，是否持久分配内存空间储存HashTable
