PHP代码及命名规范

发表于 2013-05-16   |   分类于 PHP   |  

一个优秀的团队代码规范是必须的，可以促进团队合作，降低维护成本，体现团队的凝聚力，直接有助于代码审查，对于个人来说，有助于自身的成长。整理了现有团队的代码规范，虽不是最标准，但是收集了很多优雅规范。

基本规范

文件

  • 所有文件，保存格式为UTF-8
  • 对于只包含PHP代码的文件，PHP的结束标记?>，不允许存在
  • 使用制表符TAB，4个字符宽度缩进，并且在编辑器中设置4个空格代替TAB
  • 每行保持大约80-120个字符长度

关于空格

  • 逗号后面紧跟一个空格
  • 操作符两个加空格
  • 函数关键字与左括号“(”紧贴在一起，除此之外包括结构关键字等与“(”之间都应该加一个空格

  • 右括号“)“除后面是“)”、“.”、“;”以外，其它一律用空格隔开它们

  • 单行注释“//”后紧跟一个空格

  • 程序中除字符串外不应有其它连续空格

  • 每行结尾不允许多余的空格

  • 赋值操作符前后一个空格

换行

当一个表达式无法容纳在一行时（最多120字符，最好不要超过80），依据以下原则断开：

  • 在一个逗号后面断开

  • 在一个操作符前面断开

  • 新的一行应该与上一行同一级别表达式的开头处对齐

  • 业务逻辑分块，如果在一个方法中完成一个小事务可以用换行分隔开其他事务

引号

  • PHP中单引号和双引号具有不同的解析，单引号中，任何变量($var)、特殊转义字符(“\t \t \n”)不会被解析，因此PHP解析的更快，转义字符仅支持”\’“和”\”；双引号中，能被解析的都被解析，PHP常量不能被解析。

  • 字符串是HTML代码的使用单引号

  • 正则表达式使用单引号

  • 包含有` ‘ 的字符串, 使用双引号引起来，例如SQL语句.

  • 不需要双引号的地方一律用单引号

  • 数组中下标中不是常量或者变量的使用单引号，如果数组使用在双引号字符串中，用{}包裹，或者不加引号

对齐

如果同一个内容要换行或有多个值，一般以操作符为左边界进行对齐。如：

  • 数组
  • 双元运算符? ：的对齐
  • SQL语句的对齐

控制结构

  • 不要把小括号和关键词紧贴在一起，要用空格隔开它们

  • 小括号和函数名要紧贴在一起

  • 结构体开始“{“，如果是函数、类声明时独占一行，其他不占一行而是于空格隔开紧跟空格后面；整个结构结尾“}”都独占一行

  • 即便if后只有一条语句，仍然需要加入大括号，以保证结构清晰

命名约定

  • 以标准计算机英文为蓝本，杜绝一切拼音、或拼音英文混杂的命名方式

  • 命名必须足够长以使他人能够快速理解其要表达的含义，避免使用生癖词汇、缩写过简的命名，遵循顾名能思义原则

  • 常量一律大写，多个单词以下划线“_”分割

  • 变量名或类的属性名、函数名、类方法名一律采用驼峰命名

  • 类名首字母大写，驼峰命名

  • 类成员非公开时必须以private、protected声明，并以下划线“_”开头

SVN规范

基本要求

在SVN的日常更新中我们做了如下要求：

  • 每次提交必须填写注释，注释不能少于5个中文字符，注释不能毫无意义，要讲明白本次提交的问题。

  • 一次提交只解决一个问题，不能一次性提交多个模块/问题的更新。

  • 不要尝试无意义的提交。

  • SVN提交注释填写标识说明：

    1. * 表示更新功能

    2. + 表示增加文件或功能

    3. - 表示删除文件或功能

示例

解决抓取页面缓慢的问题，注释则应填写为：

* 解决抓取页面缓慢的问题，原因是：header中设置的cookie不合理

新增了一个HTTP请求处理类，注释则应填写为：

+ 类库中新增了一个HTTP请求处理类

删除了一个已经不再使用的文件，注释则应填写为：

- 删除文件ufiles.php，下线功能u盘，删除相应文件

框架规范

我们基于Yaf的程序框架，有一些特殊的规范，也是最重要的规范。

使用命名空间

我们采用PHP-5.4.31的PHP版本，支持命名空间，程序中所有涉及到类的加载都采用命名空间的方式完成按需加载。如有不了解命名空间的同学，请参考：http://php.net/manual/zh/language.namespaces.php

use关键字

在编码中对use关键字的使用，不做强制规定，即下面两种写法均可以接受

目录与文件名

所有目录及文件名首字段大写，多个单词按照驼峰规则（其中对于Views下面的与控制器名称对应的视图文件目录及视图文件名小写）

类名与文件名

本地类库与全局类库类名与文件名保持一致，一个文件只允许定义一个类，对于特称的Yaf框架中的类如：控制器、插件、模型等，在文件名中不带相应的后缀。

Controller

对应的控制器文件名为：Index.php

Plugin

对应的插件文件名为：ModuleInit.php

Model

对应的模型文件名为：Member.php

throw Exception

对应可以重试或者自行捕获的异常，在业务逻辑代码中进行捕获，对于流程不可以走通的异常，直接抛出交由全局来进行捕获并写入日志。

视图文件

对于视图文件是PHP的脚本，那么视图只完成变量的输出及简单的遍历，业务逻辑不能在视图文件中完成。采用PHP的短标记（short_open_tag = On），注意只在视图文件才使用短标记。

变量输出

<?=变量名?>

控制结构与遍历

采用PHP语言控制结构的替代语法

PHP 提供了一些流程控制的替代语法，包括 if，while，for，foreach 和 switch。替代语法的基本形式是把左花括号（{）换成冒号（:），把右花括号（}）分别换成 endif;，endwhile;，endfor;，endforeach; 以及 endswitch;，不清楚的同学参考：http://php.net/manual/zh/control-structures.alternative-syntax.php

输入与输出过滤

为了防止XSS的攻击，我们在表单输入进行了全局过滤，在Yaf请求对象Yaf\Request\Http中获取的参数与使用$_GET/$_POST获取参数效果相同，针对PHP的全局GPRC，我们在InputFilter插件中自动完成过滤，可以直接使用。如果变量输入已经过滤，变量的输出理论上不会有问题，具体情况待研究。
